Moneyball

  • Category: Desktop App
  • Technologies: Python, Tkinter, Pandas, Multithreading
  • Project date: November, 2020

Use Case

This app was developed for a part-time financial trader. The client traded in volatile derivative contracts and was exposed to the risk of adverse price movement when he was away from the trading desk. This application solves that problem for him by monitoring prices and automatically hedging his open positions when the market moves against him.

Design

In lieu of its multi-threaded design, the various modules of this app run on their separate threads. The app has the following modules:

  1. User Interface: The UI is powered by the Tkinter library. The app's main thread is dedicated to reading user gestures on this interface. This thread executes all event listeners associated with the various UI elements like buttons and text boxes.
  2. The Rule Book: The Rule Book is a Pandas Dataframe that saves all the active rules that the system is monitoring. Each rule is basically a price condition on a particular security. The system is supposed to trade whenever the price of that security meets that condition. When a condition is met, the trade corresponding to that rule is placed and that rule is moved to the Execution Book. A separate thread monitors the execution status of that trade in the Execution Book. When the trade is successfully executed at the stock exchange, that thread moves that rule to the Open Book. A third thread monitors the rules in the Open Book and checks if their exit conditions are met. When an exit condition is met, another offsetting trade is placed to exit the open position created by the first trade. Once the exit trade is placed, the rule re enters the Execution Book. When that trade is executed, that rule is finally moved back into the original Rule Book. This way the cycle continues till the time the user manually deactivates a rule.
  3. Stock Broker Connection: The system is connected to Zerodha via the Kite Connect APIs. The system receives real-time price updates, places orders and monitors execution status of these orders via these APIs.
